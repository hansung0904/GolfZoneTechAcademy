01. 데이터베이스 기본개념
    - 데이터 정의 : 데이터는 현실 세계에서 관찰되거나, 측정되는 모든 값을 의미
    - 데이터 베이스 정의 : 데이터베이스는 특정 조직의 여러 사용자가 데이터를 공유, 사용할 수 있도록
                          통합 저장된 데이터의 집합이다. 행과 열로 구성된 시트에서 사용자가 정의한 형식으로
                          데이터를 관리하는 엑셀과 유사하다.

    - 데이터베이스 특성 : 데이터베이스는 일반적으로 다음과 같은 특성을 가지고 있다.
    1. 데이터 공유 : 여러 사용자가 동시에 사용할 수 있도록 공유한다.
    2. 데이터의 지속적인 변화 : 데이터베이스에 저장된 데이터는 지속적으로 변화한다.
    3. 데이터 값 참조 : 데이터가 저장된 저장 장치의 주소가 아닌 데이터의 값을 반환한다.
    4. 실시간 접근성 : 사용자의 데이터 요구에 대해 실시간으로 응답한다.

    - 데이터베이스 관리시스템 정의
        : 데이터베이스 관리시스템(Datebase Management System - DBMS)
        은 데이터 베이스를 생성 및 제어하고, 사용자의 요구에 따라 데이터를 처리하는 프로그램

    RDBMS의 종류 : MySQL, MariaDB, ORACLE DB, MSSQL 등등

02. 요구사항 수집 및 분석
    - 프로그램 개발과정
    - 프로그램은 일반적으로 다음과 같은 과정으로 개발된다.
    -> 서비스 기획 -> 요구사항 수집/분석 -> 프로그램 설계 -> 프로그램 구현


    - 요구사항 수집 및 분석 정의
        - 데이터베이스 설계에 필요한 다양한 정보와 요구사항을 수집 및 분석하는 작업을 의미
        - 정확한 데이터베이스 설계의 기반이 됨.
        - 만약, 요구사항 분석이 잘못된 경우 관련된 모든 프로그램에 영향을 미치므로
        - 꼼꼼하게 분석할 필요가 있다.

    - 요구사항 수집방법
        - 업무 관련문서 , 사용자 면담 , 사용자 설문조사    

03. 데이터 모델링 개요
    - 데이터 모델링 정의
        - 데이터 모델링은 현실 세계에서 수행하는 업무를 이해하고 분석한 후, 의사결정을 통해 데이터의 구조와
          규칙이 표현된 데이터 모델을 생성하는 과정이다.
        - 데이터베이스 설계의 핵심 단계이다.

        현실 세계의 업무 -> 업무 분석 -> 데이터 모델          

    - 데이터 모델링 목적
        - 데이터 모델링은 업무에 필요한 데이터를 일관되고, 정확하게 관리하기 위해 수행한다.
        !!! 설계는 정확한 결과물의 기분이 된다.    

    - 데이터 모델링 단계
        - 데이터 모델링은 개념 데이터 모델링, 논리 데이터 모델링, 물리 데이터 모델링 단계로 진행
        - 각 단계의 데이터 모델링마다 개체-관계 다이어그램 (ERD)으로 표기한 데이터 모델을 생성한다.
        -> 개념데이터 모델링 -> 논리데이터 모델링 -> 물리데이터 모델링 -> 데이터베이스 구현

    - 개념 및 논리 데이터 모델 구성요소
        - 개념 및 논리 데이터 모델은 엔티티, 속성, 식별자 및 관계로 구성되어 있다.

        Entity : 데이터 저장의 대상이 되는 유무형의 개체 및 행위에 대한 데이터 집합
        Attribute : Entity에서 관리하는 구체적인 정보 항목으로, 데이터 집합의 특성을 나타냄
        Identifier : Entity 내 각각의 인스턴스를 구분할 수 있는 구분자로서, 모든 인스턴서의 유일성을 보장
                     식별자는 하나 이상의 속성으로 구성될 수 있음.
        Relationship : 업무와 관련된 Entity들이 어떻게 연관되어 있는지를 표현
                       데이터의 참조 무결성을 보장하기 위한 업무 규칙.

    - Entitiy
        - 데이터 저장의 대상이 되는 유무형의 개체 및 행위에 대한 데이터의 집합을 의미

    - Attribute 
        - Entitiy에서 관리하는 구체적인 정보 항복으로, 데이터 집합의 특성을 나타냄.

    - Identifier
        - 식별자는 Entity 내 각각의 인스턴스를 구분할 수 있는 구분자로서, 모든 인스턴스의 유일성을 보장.
